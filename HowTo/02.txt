In the first howto, we did a lot of setup work.

we.
1. created a python virtual environment
2. installed django into that environment.
3. started a django project named "cs50"
4. ran the test server using "python manage.py runserver"
5. checked the output from the server using a webbrowser pointed at http://127.0.0.1:8000
6. put the project under git version control

that output informs us that the database used by the project has "unapplied
migrations" that is a hint as to why db.sqlite3 is an empty file.


if we run the command suggested. (again after activating the virtualenv)

(venv) me@my-comp ~/project/cs50 $ python manage.py migrate

then that db.sqlite3 file gets populated with a schema.


using the skills learned in cs50 week7, we can examine that schema.

I will leave this here for the interested.
Let's commit this file to git, and then start to modify the django project
itself.

reminder for git
me@my-comp ~/project/cs50 $ git add .
me@my-comp ~/project/cs50 $ git commit -m "db.sqlite3 has changed after running migrate"
