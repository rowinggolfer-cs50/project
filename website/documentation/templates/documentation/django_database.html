The Django Database.

04.html

Django wesbites all have a database component.
Django can use many database engines (MariaDB, MySQL, PostgreSQL etc...)
However, by default, Django uses an sqlite3 database. This is convenient for simple projects.

Previously when we ran the test server for the first time that output informed us that the database used by the project has "unapplied migrations" and sugested we ran "python manage.py migrate".
The implication is that the database "db.sqlite3" which has been created in our project directory is empty.

If we run the command suggested. (again after activating the virtualenv) 

(venv) me@my-comp ~/project/cs50 $ python manage.py migrate

Now our db.sqlite3 file gets populated with a schema.
Using the skills learned in cs50 week7, we can examine that schema.
I will leave this here for the interested. 

 
(venv) me@my-comp ~/project/cs50 $ sqlite3 db.sqlite3
SQLite version 3.29.0 2019-07-10 17:32:03
Enter ".help" for usage hints.
sqlite> .schema
