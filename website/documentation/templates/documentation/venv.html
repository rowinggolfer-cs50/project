Virtualenv

venv.html

Create a virtualenv for python.
This is sensible as it helps keep your system's python environment clean. A virtualenv is created by default without any access to site-packages and other locally installed modules. This is useful.
This also ensures you are aware what third party modules you have used, which is essential for deployment of your django project to a server

examples in creating a virtualenv

(choose which python to "clone" ) 

me@my-comp ~/project $ virtualenv venv
OR
me@my-comp ~/project $ virtualenv -p python3 venv
OR
me@my-comp ~/project $ virtualenv -p /usr/local/bin/python3.8 venv

now the project directory looks like this. (tree is a nice tool for seeing directory structures. -L flag indicates depth) 

me@my-comp ~/project $ tree -L 2
.
└── venv
    ├── bin
    ├── include
    ├── lib
    └── share

start your virtual environment and check that the correct python is in use. 

me@my-comp ~/project $ source venv/bin/activate

(venv) me@my-comp ~/project $ which python
me@my-comp ~/project/venv/bin/python

Note

  • "(venv)" reassures us that we are in a python virtual environment.
  • the executable returned by "which python" confirms it.
