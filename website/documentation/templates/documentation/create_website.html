Let's just take a look at what django has provided out of the box.

Note: We have 2 folders called cs50.
cs50 is simply the PROJECT_NAME I chose when running 

 django-admin startproject PROJECT_NAME
having 2 such folders is good design, which will become clear one we start our "apps", however it does lead to some unavoidable ambiguity in this documentation.
I will differentiate by referring to the parent directory as "root cs50".

Files and Subfolders in "root cs50"

in the "root cs50" folder, we have 2 files and a folder.
SubFolder - cs50

in this "project cs50" folder, we have 5 files.

__init__.py
this file is blank - but confirms that this directory is an importable python package
asgi.py and wsgi.py
These files are used for deploying to webservers. wsgi has been around for years. asgi is a new emerging standard (with with I am unfamiliar). We don't really need to touch these files, although it can be useful to put code therein to confirm you are in a deployed environment and overwrite some settings. But we are not at that stage yet, so we can leave them alone!
urls.py
out of the box, django has one url pattern in this file.
urlpatterns = [path('admin/', admin.site.urls),]
urls.py is where we build our hyperlinks for the website. this default file will catch http://127.0.0.1:8000/admin/
settings.py.
This file is extremely important, and must be configured correctly for oue new website to function correctly and securely.

File 1 - db.sqlite3

Our sqlite3 database. For a small website, sqlite is good fit, as it makes for very easy deployment. django can use a variety of industry standard databases such as postgresql, mariadb and mysql, but that is beyond the scope of this project.
- more details about how django uses databases and sqlite3
